12.3. Font Animations

12.3. Font Animations

Font animations define a font to be used by text animations. Font animations are not actually meant to be used by objects themselves, and nothing will display if they are used. Fonts function similarly to sprite animations, they specify a sprite to use for each ASCII character. The first sprite describes the character 0x20 in the encoding, which corresponds to a space character, and each following sprite describes the next character, and so on. NTSC-J versions of the games have special behavior for character 128 and onwards, so upwards of 96 characters may be defined in a single font. Font animations are type 3.

The format for a font animation is as follows: #font anim-name tpag-eid anim-name is an identifier for the animation's name. tpag-eid is the EID of the texture page that contains the sprite(s).

This is then followed by a list of character definitions: #char rgb color-mode blend-mode tex-x tex-y clut-x clut-y tex-w tex-h char-w char-h #char rgb color-mode blend-mode tex-x tex-y clut-x clut-y tex-w tex-h ! flip wind char-w char-h rgb, color-mode, blend-mode, tex-x, tex-y, clut-x, clut-y, tex-w, tex-h, flip, and wind are the same parameters as in sprite animations. char-w and char-h specify the width and height of the character, with 400 corresponding to a "normal" size. char-h can also be zero, in which case it assumes a default value.
