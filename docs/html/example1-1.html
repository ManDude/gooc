3.1.1. Object Definition

3.1.1. Object Definition

We start off the object with a GOOL declaration:

// Declare a DbgOC GOOL executable. #gool DbgOC 63 2 // Include a file we made. #include "GOOL_EXPR.gooc"

Just like in C, comment blocks are placed between /*...*/ and single-line comments are placed after //.

The #gool DbgOC 63 2 line defines the properties of our GOOL executable. It sets the name to DbgOC, the GOOL object ID (see Object IDs) to 63 and the GOOL object type (see Object Types) to 2. This line is necessary because the name, ID and type fields are mandatory. If this line is not present, the compiler will only parse our code but not compile it. This line will also create a new expression macro with the name DbgOC (our name) with value 63 (our ID), so that it can be used for code that requires object ID, such as spawning children objects.

The #include directive includes the file GOOL_EXPR.gooc (which is in the same folder as our main file) verbatim. This is a file that includes a bunch of expression macros and inline functions which will come in handy when making our GOOL objects.
