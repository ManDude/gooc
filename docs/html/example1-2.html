3.1.2. Animations

3.1.2. Animations

Here are the animation definitions that we will use for our test object. Keep in mind the format of these is highly susceptible to change for each different game.

#anim TEST_ANIM [CE10V] 28 1 #anim BOX_ANIM [BN10V] 1 0

These two lines both describe a new vertex animation each. The first one we named TEST_ANIM and mapped it to the animation file called CE10V - Crash's explosion death animation. We also say that it has 12 frames of animation and that the animation should have 1 frame of interpolation (this will squeeze in an extra frame of animation inbetween every stored frame). Each game has a slightly different format for these types of animations. See Vertex Animations.

The second one is named BOX_ANIM, mapped it to animation BN10V, say that it has 1 frame of animation and that it should not be interpolated (which requires at least 2 frames to work anyway).
