3.1.3. Spawns

3.1.3. Spawns

In order for an object to appear it needs to be spawned, and GOOC uses spawn definitions to determine how to spawn a certain object. See Spawns. You can compare an object's spawn field against a spawn in order to run some spawn-specific code without having to write multiple states. For this example we will only need one spawn.

#spawn S_ROT_TEST Rot_Test_Main

Our spawn is named S_ROT_TEST and its spawn state is Rot_Test_Main.
