6.2. Arrays

6.2. Arrays

An array is a data structure that lets you store one or more elements consecutively in the GOOL data pool. Array elements are indexed beginning at position zero, not one.

To define an array, you need to specify the data type array, its name, and then the list of elements, separated by commas, in a set of braces. Array elements must be integers, and cannot be EIDs. Once defined, arrays are merely an index into the GOOL data pool where the elements are stored, meaning they are not valid addresses that can be written to. array MyArray1 = { 1, 2, 4, 8, 16 }

Arrays are data structures meant to be passed as a parameter for certain instructions, and are usually not meant to have their elements directly accessed inside source code. In Crash 1, the specialized dereference instruction must take a constant element index number. There is never bounds detection for arrays, nor is their size stored anywhere.
